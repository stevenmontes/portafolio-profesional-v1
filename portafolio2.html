Proyecto 2

Este proyecto se trata de vender productos, incluso puede solicitar los productos y los vendedores pueden ofrecer sus productos.

El cliente principal del proyecto, eran varios tipos de usuarios, habían administradores, proveedores, distribuidores, tiendas y clientes.

Los clientes son la mayor prioridad, se encarga de comprar productos de las tiendas y también puede realizar subastas de los productos y las tiendas les ofrencen los productos.

Además, los clientes podían registrar direcciones, para que sus productos llegaran a su propia casa o destino.

Las tiendas, registraban sus productos, distribuidores y proveedores. Podían ganar ingresos en base a las ventas que generaban. El sistema se quedaba con un porcentaje de las ganancias.

Los distribuidores eran encargados de distribuir los productos comprados del sistema, recibían solicitudes de compra de parte de los clientes o tiendas y llevaba los productos al destino ingresado por el cliente.

Los proveedores se encargaban de ofrecer los productos a las tiendas. Los productos de las tiendas podían venir de proveedores o no.

¿Qué apredí de este proyecto?

Este proyecto aprendí a usar API REST para una aplicación web.

Se utilizó un arquitectura de sistema de N-Capas. Tenía 5 capas: interfaz gráfica, API REST, manager, crud factory y mapper.

En donde, cada capa tenía una responsabilidad, los mappers se encargaba de mapear los querys de SQL y con la comunicaciòn de la base de datos. Los crud factory, se encargaba de construir los objetos, despúes de recibir la información del mismo.

Los Manager se encargaban de gran parte de la lógica del sistema. El API REST devolvía los objetos en respuesta de las solicitudes de la interfaz gráfica.

Se construyó el proyecto en la herramienta Visual Studio, con el lenguaje de programación C#. Se utilizó varias librerías para el manejo de errores y correos electrónicos.

También se construyó una interfaz gráfica web con .NET, que se alimentaba del API REST, y para la administración de la base de datos, se usó SQL SERVER.

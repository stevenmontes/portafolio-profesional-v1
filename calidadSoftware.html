La importancia de las pruebas unitarias.

Las pruebas unitarias o Unit Testing son importantes en la hora de desarrollar un software porque es la forma de asegurarse que nuestro sistema funciona correctamente.

Un software sin pruebas unitarias son una bomba a punto de estallar.

¿Qué son las pruebas unitarias?

Las pruebas unitarias son segmentos de códigos diseñados para comprobar el código principal esta funcionando como se espera.

Tipos de pruebas unitarias

Hay varios tipos de pruebas:

  • Test unitarios: Prueba una funcionalidad en concreto y se basan en el principio de una sola responsabilidad.

  • Pruebas de integración: prueba la conexión entre componentes.

  • Pruebas funcionales: Prueban la integración de todos los componentes que desarrollan una funcionalidad concreta (por ejemplo, la automatización de pruebas con Selenium serían test funcionales).

  • Pruebas de aceptación de usuarios: Pruebas definidas por el dueño del productos

  • Pruebas de regresión: Prueban que los test unitarios y funcionales siguen funcionando a lo largo del tiempo.

  • Pruebas de carga: Prueban la eficiencia del código.

Características de una buena prueba unitaria

Las pruebas unitarios deben cumplir las siguientes características:

  • Rápida No es infrecuente que los proyectos maduros tengan miles de pruebas unitarias. Las pruebas unitarias deberían tardar muy poco tiempo en ejecutarse.

  • Aislada Son independientes, se pueden ejecutar de forma aislada y no tienen ninguna dependencia en ningún factor externo.

  • Reiterativa La ejecución de una prueba unitaria debe ser coherente con sus resultados, es decir, devolver siempre el mismo resultado si no cambia nada entre ejecuciones.

  • Autocomprobada La prueba debe ser capaz de detectar automáticamente si el resultado ha sido correcto o incorrecto sin necesidad de intervención humana.

  • Oportuna Una prueba unitaria no debe tardar un tiempo desproporcionado en escribirse en comparación con el código que se va a probar. Si observa que la prueba del código tarda mucho en comparación con su escritura, considere un diseño más fácil de probar.

Algunas buenas ventajas de usar pruebas unitarias

  • Calidad del código Al realizar pruebas continuamente y detectar los errores, cuando el código está terminado, es un código limpio y de calidad.

  • Detectar errores rápido Las pruebas unitarias nos permiten detectar los errores rápidamente, haciendo pequeñas pruebas y de manera periódica.

  • Proporciona información Se puede recopilar gran cantidad de información gracias al continuo flujo de pruebas y así se evita errores venideros.

  • Facilita los cambios y favorece la integración Los pruebas unitarias, nos permiten modificar partes del código sin afectar al conjunto, simplemente para poder solucionar errores que nos encontramos por el camino.

  • Si primero se crean los pruebas, es mucho más fácil saber con anterioridad cómo debemos enfocar el diseño y ver qué necesidades debemos cumplir al crear el código. Este es una técnica de desarrollo llamada TDD (Test Driven Development)

La calidad en el software es sumamente importante, te asegura que las funcionalidades de un sistema estén correctos, si aplica bien las pruebas unitarios y buen manejo y organización de las pruebas unitarias, se brinda un excelente producto de software al cliente.

Y así con la calidad de software en un software, mantememos felices a los clientes.
